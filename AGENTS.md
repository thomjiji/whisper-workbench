# AGENTS.md

This repository uses GitHub Issues and the GitHub Project as the only task-tracking system.

Read order:

1. `AGENTS.md` for the short agent entry point.
2. `docs/agent-workflow.md` for collaboration rules, verification, and doc ownership.
3. `docs/architecture.md` and `docs/cli.md` for implementation and usage details.
4. The active GitHub issue or pull request for the current task.

Local notes:

- `tasks/lessons.md` stores reusable lessons from past corrections.
- `.codex/HANDOFF.md` is optional scratch context, not a backlog.
- `.codex/TASKS.md`, `.codex/TODO.md`, and `.codex/ISSUES.md` were intentionally removed after migrating planning to GitHub.
