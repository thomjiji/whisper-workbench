# Plans

This directory stores reviewable implementation plans for non-trivial GitHub issues.

Use a plan doc when the work:

- spans multiple files or modules
- changes architecture, CLI behavior, or UI behavior
- has meaningful regression risk
- needs explicit scope review before coding

Suggested filename format:

- `YYYY-MM-DD-issue-id-short-name.md`

Example:

- `2026-03-12-i-006-final-punctuation-pass.md`

Plan docs are not a second backlog. GitHub Issues and the GitHub Project remain the only task-tracking system.
